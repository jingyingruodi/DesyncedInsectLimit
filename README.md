# Desynced 虫群单位上限修改模组 (Insect Limit Mod)

这是一个专为修复原版 AI 缺陷、优化生命周期并引入活跃玩家动态权重的虫群增强模组。

## 📖 版本特性 (v2.7.2 Final)

### 1. 动态可扩张上限系统
- **动态绝对上限**：基础 12,000 只活跃战斗单位，每增加一名**存活玩家**额外 +3,000。
- **动态削弱阈值**：基础 4,000 只，随玩家数动态增长，超过后刷怪量减半，维持服务器 UPS。
- **动态扩张阈值**：基础 6,000 只，确保侦察虫在合理范围内持续建立新巢穴。
- **活跃度播报**：控制台实时显示 `Players: 存活/总数`，精准监控服务器对抗强度。

### 2. 智能扩张与导航修复
- **前线单位锁定**：侦察虫不再死板锁定玩家主基地，而是动态锁定**物理距离最近**的玩家单位进行定向扩张，有效克制“移动基地”规避战术。
- **超视距导航**：解锁了侦察虫的寻路限制，支持跨越 700+ 格的超远距离渗透。
- **180s 侦察虫容错**：侦察虫卡死处决时间延长至 180 秒（900 ticks），大幅提升其绕过长城或复杂地形建立前线蜂巢的成功率。

### 3. 生命周期与性能优化
- **侦察虫高效精简**：超时自毁且拦截归巢尝试，绝不占用宝贵的蜂巢坑位，确保兵力配额全部分配给战斗单位。
- **归巢死循环修复**：当归巢路径被山崖或长城阻断时，该巢穴会被拉黑 250 秒，强制虫子寻找其他出路。
- **30s 极效普查**：采用 150 ticks 低频缓存统计，兵力计数开销仅为 O(1)。
- **目标合法性校验**：底层过滤蓝图、矿点、中立遗迹，彻底解决单位堆积 Bug。

### 4. 生产级稳定性
- **全量覆盖**：针对关键生成逻辑进行主权级重写，防止原版隐藏变量干扰。
- **中英双语注释**：代码内含详尽注释，方便接手与维护。

## 🛠 安装方法
1. 将模组文件夹放入游戏的 `Content/mods/` 目录下。
2. 在模组菜单中启用 `InsectLimit`。
