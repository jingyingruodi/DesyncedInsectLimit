# Desynced 虫群单位上限修改模组 (Insect Limit Mod)

这是一个专为修复原版 AI 缺陷、优化生命周期并引入活跃玩家动态权重的虫群增强模组。

## 📖 版本特性 (v2.7.15 Final Stable)

### 1. 行为全局解耦限速系统 (New!)
- **三路独立冷却**：将虫群的“派遣侦察”、“发起进攻”与“自然扩张”三项行为的全局冷却时间（Cooldown）完全分离。
- **并行决策**：不同蜂巢可以同时执行不同的任务，不再因全球共用一个计时器而导致反应迟钝。
- **存活玩家动态权重**：频率限制随服务器内**存活玩家数量**实时缩放，玩家越多，虫群活动越频繁。

### 2. 动态可扩张上限系统
- **动态绝对上限**：基础 12,000 只活跃战斗单位，每增加一名**存活玩家**额外 +3,000。
- **动态削弱阈值**：基础 4,000 只，随玩家数动态增长，超过后刷怪量减半，维持服务器 UPS。
- **活跃度播报**：Log后台实时显示 `Players: 存活/总数 | BOTS: 当前/上限`，精准监控对抗强度。

### 3. 智能扩张与导航修复
- **前线单位锁定**：侦察虫不再死板锁定玩家主基地，而是动态锁定**物理距离最近**的玩家单位进行定向扩张，克制“移动基地”规避战术。
- **超视距导航**：解锁了侦察虫寻路限制，支持跨越 700+ 格的超远距离渗透，并具备 180 秒（900 ticks）卡死容错。
- **归巢死循环修复**：当归巢路径被地形阻断时，该巢穴会被拉黑 250 秒，强制虫子寻找其他出路。

### 4. 生命周期与性能优化
- **精准战斗判定**：重构了卡死检测逻辑。只有在单位真正进行攻击或生命值受损时才重置卡死计时，彻底解决“残血发呆”单位永久占坑的问题。
- **目标合法性校验**：底层过滤蓝图、矿点、中立遗迹，彻底解决单位在无效目标点堆积的 Bug。
- **30s 极效普查**：采用 150 ticks 低频缓存统计，确保大规模单位下的性能开销降至 O(1)。

## 🛠 安装方法
1. 将模组文件夹放入游戏的 `Content/mods/` 目录下。
2. 在模组菜单中启用 `InsectLimit`。
