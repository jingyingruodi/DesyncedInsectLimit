# Desynced 虫群单位上限修改模组 (Insect Limit Mod)

这是一个专为修复原版逻辑缺陷并提升多人服务器体验而设计的虫群上限提升优化模组。

## 📖 模组背景 (Background)
在《Desynced》原版逻辑中，虫群存在多项 AI 缺陷（如围攻蓝图、千里奔袭等），且全图单位上限过低（4,000只）。在多人模式下，由于所有玩家共用上限，导致大后期虫群活跃度极低，严重影响防守玩法的乐趣。

## ✨ 核心功能 (Features)
本模组 v2.5.2 版本基于最稳定的逻辑骨架重构，实现了性能与智能的完美平衡：

- **AI 顽疾修复**：
  - **目标合法性校验**：彻底解决虫群围攻“建筑蓝图”、“可探索遗迹”或“掉落物”导致单位堆积的 Bug。
  - **隐身基地适配**：当玩家主基地隐身时，虫群会自动转为攻击附近可见的活跃单位，而非原地发呆。
  - **距离截断机制**：修复原版“千里奔袭”Bug。若目标过远（>250格），虫群将就近打击周边玩家资产，拒绝无效的长途行军。

- **性能极限优化**：
  - **30s 全局普查心跳**：采用极致低频的统计缓存机制，实现 O(1) 级兵力计数开销，彻底告别大规模存档下的逻辑掉帧。
  - **采样目标搜索**：优化蜂巢索敌算法，采用随机采样机制代替全量扫描，显著降低服务器负担。

- **规模与配额优化**：
  - **绝对上限提升**：上限提升至 **30,000** 只活跃战斗单位。
  - **生成波次增强**：将生成数量开始缩减的阈值从原版的 2,000 提升至 **4,000**。
  - **扩张意图放宽**：派遣侦察兵建立新巢穴的判定阈值提升至 **10,000** 只单位。
  - **纯单位统计哲学**：上限**仅计算活跃战斗单位**，虫巢建筑不再占用刷怪名额，确保扩张后的进攻烈度不减。
  - **病毒自动回收**：完善了底层受感染单位的关机与覆灭倒计时机制，自动清理无效实体。

- **服务器完美兼容**：纯逻辑修改，支持专用服务器（Dedicated Server）及自动同步。

## 🛠 安装方法 (Installation)
1. 下载本项目文件。
2. 将项目文件夹（或打包好的zip压缩包）放入游戏的 `Content/mods/` 目录下。
3. 启动游戏并在模组菜单中启用。

## ⚠️ 注意事项
- **TPS 说明**：本模组针对游戏 5 TPS (每秒5次Tick) 环境进行了精确时间适配。
- **性能提示**：30,000 单位为理论上限，实际压力取决于服务器 CPU 性能。
