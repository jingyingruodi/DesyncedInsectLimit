# Desynced 虫群单位上限修改模组 (Insect Limit Mod)

这是一个专为修复原版逻辑缺陷、优化生命周期并引入动态上限而设计的虫群增强模组。

## 📖 模组背景 (Background)
在《Desynced》原版逻辑中，虫群在大后期往往面临两个极端：要么因为统计口径错误（将虫穴计入上限）导致不再刷怪，要么因为无效寻路（千里奔袭）导致服务器掉帧。本模组旨在解决这些痛点，同时保持原汁原味的扩张体验。

## ✨ 核心功能 (Features)

- **动态可扩张上限 (v2.6.2 新增)**：
  - **动态绝对上限**：基础 18,000 只，每增加一名玩家额外 +4,000。
  - **动态削弱阈值**：基础 6,000 只，随玩家数动态增长，超过后自动降低刷怪烈度。
  - **动态扩张阈值**：基础 9,000 只，确保侦察虫在合理范围内持续建立新巢穴。

- **侦察虫生命周期管理**：
  - **卡死即处决**：侦察虫若因玩家长城或复杂地形卡死 60 秒，将直接被销毁，不再无限期尝试寻路。
  - **拦截归巢**：侦察虫一旦进入“无主”状态将立即自毁，绝不占用宝贵的蜂巢坑位，确保兵力配额全部分配给战斗单位。

- **AI 顽疾修复**：
  - **目标合法性校验**：彻底解决虫群围攻“建筑蓝图”、“矿点”或“中立遗迹”导致的单位堆积。
  - **250 格性能截断**：将单次入侵的寻路范围限制在 250 格内，大幅降低大型存档下的逻辑计算开销。
  - **对齐原版扩张感**：侦察触发距离恢复为 100 格，确保虫群扩张节奏与原版一致。

- **性能极限优化**：
  - **30s 全局普查心跳**：采用 150 ticks 低频统计缓存，实现兵力计数的高性能 O(1) 访问。
  - **纯单位统计哲学**：上限**仅计算活跃战斗单位**，虫巢建筑不再占用名额。

## 🛠 安装方法 (Installation)
1. 将模组文件夹放入游戏的 `Content/mods/` 目录下。
2. 在模组菜单中启用 `InsectLimit`。
